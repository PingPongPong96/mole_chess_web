# 鼹鼠棋 Mole Chess Web

纯前端本地规则引擎版本（`index.html + app.js + local_api.js + game_core.js`）。

## 运行

1. 主菜单入口：直接打开 `portal/main_menu.html`。
2. 或直接打开 `index.html`。
3. 点击“开始游戏”，完成先手骰。
4. 常规模式下通过右键菜单行动；沙盒模式可放开摆位。

单文件打包：

```bash
pip install beautifulsoup4
python build_portable.py
```

输出：`mole_chess_portable.html`

## 代码结构

- `game_core.js`：核心规则、棋子类、回合推进、AI、序列化。
- `local_api.js`：本地 `/api/*` 接口适配与状态快照。
- `app.js`：前端交互、渲染、动画、技能流程。
- `style.css`：界面样式。
- `piece_data.js`：棋子规则说明文案。
- `ai/engine_v2.js`：AIEngineV2（搜索优先、机会节点、确定性）。
- `ai/worker/ai_worker.js`：AI Worker 入口。
- `ai/ai_config_store.js`：AI 配置存储（`mole_chess_ai_config`）。
- `ai/ai_telemetry_store.js`：AI 遥测存储（`mole_chess_ai_telemetry`）。
- `ai/models/`：模型与 `manifest.json`。
- `ai/runtime/`：端侧推理 runtime（当前含占位文件）。
- `ai_training/`：Python 训练/导出/评测脚手架。
- `tests/ai_snapshots/`：一致性快照集。
- `tests/ai_consistency_runner.py`：跨端一致性评测脚本（Playwright）。

## 本次全局修复（已生效）

### 1. 叶某改为被动自动变身

- 移除叶某主动“变身技能”入口。
- 变身条件改为实时判定：
  - 妻子当前为绿叶妻；
  - 孩子当前已学红歌（红叶儿）；
  - 叶某到达敌方阵地。
- 满足条件时自动立刻变身夜魔，不再等待手动点技能。

### 2. 夜魔被拘留后可再次变身

- 警察拘留释放夜魔后，完整恢复为叶某状态并遣返初始位。
- 后续若再次满足三条件，可再次自动变身夜魔。

### 3. 夜魔免疫常规吃子

- 夜魔不能被任何常规吃子吃掉（包括妻子/市民/官员/老师等）。
- 夜魔仅可被：
  - 警察技能抓捕/枪决；
  - 死神机制处理。

### 4. 绿叶妻“组合堆叠棋子”强约束

- 绿叶妻 = 妻子附身市民的组合体。
- 被附身市民始终底层，绿叶妻始终顶层显示。
- 被附身市民不可独立操控，不可独立吃子，移动与状态与绿叶妻联动。
- 引擎新增组合体一致性修复器，持续保证堆叠不变量。

### 5. 绿叶妻随机移动机制修复

- 常规模式下绿叶妻移动为一键随机判定（非选格移动）：
  - 第一次骰：方向；
  - 第二次骰：步数。
- 被阻挡时停在阻挡前一格。
- 方向骰为 `0`：立即解除附身，妻子回初始位（不再掷第二骰）。

### 6. 孩子翻墙改为纯被动

- 移除“翻墙”主动技能入口。
- 翻墙跨墓为被动天赋，不摇骰。
- 学会红歌后失去该被动（保持既有规则）。

### 7. 与上轮 4 项修复兼容保持

以下行为保持不回退：

- 技能前置失败不吞回合（`action_consumed=false`）。
- 医生单墓碑复活（前后两排内选1墓）。
- 警察抓捕仅限技能射线范围。
- 技能冷却禁用态与回合结束逻辑保持一致。

## 本轮新增修复（4项）

### 1. 僧侣修行改为固定 3 回合 + 单目标锁

- 取消随机 1/3 回合与永久出家分支，改为固定 3 回合修行。
- 僧侣同一时间只能存档 1 个棋子；目标回原位前不可再次施放。
- 修行中的棋子不可移动、不可使用技能。

### 2. 全技能统一禁止以死神为目标

- 后端 `use_skill` 增加统一硬校验：技能目标格含死神即失败，且不吞行动。
- 前端选点阶段同步拦截，避免误点提交。
- AI 技能动作生成同步过滤死神目标。

### 3. 广场舞大妈技能修复

- 技能目标限制为非中立棋子（死神/鼹鼠/僧侣/魔笛手/广场舞大妈等均不可选）。
- 流程改为：先拉目标到大妈格，再按方向骰随机飞行。
- 共舞叠层吃子改为：若顶层是大妈，常规吃子可向下命中首个合法可吃目标；大妈本体不被常规吃子。

### 4. AI 升变改为回合开始强制自动决策

- AI 回合开头若存在可升变市民，先执行升变，再进行后续回合流程。
- 升变类型由后端评估打分自动选择，并用固定规则打破平局，保证可复现。

## 普通模式 vs 沙盒模式

- 普通模式：完整规则对局。
- 沙盒模式：仅放开摆位（`/api/sandbox/relocate`），其余核心机制（技能规则、抓捕范围、冷却、组合堆叠约束等）保持一致。

- `released_possession`
